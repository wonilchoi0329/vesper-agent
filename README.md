# Vesper — Design Engineering Agent

Vesper is a design engineering agent for building pixel-perfect UI from Figma. It enforces a strict token-first, registry-first workflow that prevents hardcoded values and component drift.

## Quick Start

**One-liner install:**
```bash
curl -fsSL https://raw.githubusercontent.com/wonilchoi0329/vesper-agent/main/install.sh | bash
```

Then summon with `@vesper` in Claude Code.

## Installation

```bash
curl -fsSL https://raw.githubusercontent.com/wonilchoi0329/vesper-agent/main/install.sh | bash
```

### Via agentSources

Add to your project's `.claude/settings.json`:

```json
{
  "agentSources": [
    "https://github.com/wonilchoi0329/vesper-agent"
  ]
}
```

### Manual

```bash
git clone --depth 1 https://github.com/wonilchoi0329/vesper-agent.git /tmp/vesper
cp -r /tmp/vesper/.claude /tmp/vesper/.vesper .
rm -rf /tmp/vesper
```

## How it works

```
┌─────────────────────────────────────────────────────────────────┐
│  .claude/agents/vesper.md                                       │
│  Agent definition — persona, judgment, hard rules               │
├─────────────────────────────────────────────────────────────────┤
│  ┌─────────────────┐  ┌─────────────────┐  ┌─────────────────┐  │
│  │ .vesper/        │  │ .vesper/        │  │ .vesper/        │  │
│  │ workflows/      │  │ workflows/      │  │ workflows/      │  │
│  │ bootstrap.md    │  │ figma.md        │  │ code.md         │  │
│  │                 │  │                 │  │                 │  │
│  │ One-time setup  │  │ Build from      │  │ Build without   │  │
│  │ Tokens + config │  │ Figma spec      │  │ Figma           │  │
│  └─────────────────┘  └─────────────────┘  └─────────────────┘  │
├─────────────────────────────────────────────────────────────────┤
│  .vesper/reference/      Token files       shared/              │
│  design-principles.md    (you provide)     (generated output)   │
└─────────────────────────────────────────────────────────────────┘
```

**Vesper** reads the appropriate workflow based on context, enforces the rules, and refuses to write code until you confirm the build plan. Every color, spacing value, and component comes from the shared system — nothing is hardcoded or rebuilt.

## Quick start

### 1. Add your token files at project root

**primitives.json** — raw scale values (colors, spacing units, etc.)

**tokens.json** — semantic aliases referencing primitives:
```json
{
  "color": {
    "action": {
      "primary": { "$value": "{color.blue.500}" }
    }
  }
}
```

**typography.json** — font primitives + composite text styles:
```json
{
  "font": {
    "size": { "md": { "$value": 16 } }
  },
  "type": {
    "body": {
      "md": {
        "$value": {
          "fontFamily": "{font.family.sans}",
          "fontSize": "{font.size.md}",
          "fontWeight": "{font.weight.regular}",
          "lineHeight": "{font.line-height.normal}"
        },
        "$type": "typography"
      }
    }
  }
}
```

### 2. Run bootstrap

```
@vesper Set up the Figma workflow for this project
```

Vesper reads `workflow-bootstrap.md`, detects your framework, syncs Text Styles from Figma, runs the token generator, and creates `shared/project.config.json`. Bootstrap runs once.

### 3. Build screens

```
@vesper Build this screen: https://figma.com/file/...
```

Vesper reads tokens → reads registry → extracts Figma context → shows a Pre-Build Audit → waits for confirmation → builds.

```
Pre-Build Audit: Dashboard
──────────────────────────
Reusing (2):   ✓ Button  ✓ Card
Building (1):  → StatTile
Layout only:   → page wrapper

Tokens ✓  Text styles ✓

Confirm to build?
```

Nothing is written until you confirm.

## Core rules

| Rule | Correct | Wrong |
|------|---------|-------|
| Colors | `var(--color-action-primary)` | `#3B82F6` |
| Spacing | `var(--space-component-md)` | `16px` |
| Typography | `style={textStyles.body.md}` | `fontSize: 14` |
| Components | Import from registry | Rebuild existing |
| Variants | Only what Figma defines | Invented states |

## Updating tokens

```bash
node .vesper/scripts/generate-tokens.js
```

Run after any change to `primitives.json`, `tokens.json`, or `typography.json`.

**Text Styles changed in Figma:**
- New style → ask Vesper to re-sync text styles
- Renamed → re-sync, then find-replace old `textStyles.*` keys
- Values changed → update `typography.json` first, then re-sync

## Troubleshooting

| Problem | Fix |
|---------|-----|
| Missing token in audit | Add to token file, run generator, retry |
| Unmatched text style | Add `type.*` entry to typography.json, run generator, re-sync |
| Component rebuilt unnecessarily | Add `Figma name:` to registry entry |
| Spacing looks wrong | Gap = between children, padding = inside frame |
